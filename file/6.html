关于写的一些垃圾课设/毕设

  1. 找了份软件无关的工作，也算少走40年弯路？只是抽空写着玩玩…给某些校友参考下，毕竟确实没卵用…纯粹是应付任务书、水论文，不如找点实习、学点技术、打打游戏……？？个人认为干啥都比自己浪费时间写这玩意好。
  2. 不会写的像之前我自己上课时的考虑得那么多了，老师也不关注，个人感觉老师关注的优先级：论文格式 >> 任务书功能是否完成 >> 考勤 >= 答辩 >> 其它。而且答辩即使很烂，最后肯定也不会为难你的，肯定能过。老师甚至可能都不记得你程序和你怎么答的……而且写得太完善，都不改动的话，验收时重样了就麻烦了，样式表、布局、图标、字体、功能细节……，克隆过去自己改改就好。
  3. 旧的课设我应该还是有当年的报告的，校友需要的可以联系我，能找到并能够给的尽量给你们参考……要我写新的，估计暂时不行，虽然考过了软考中级，几个破图肯定还是会画的，论文也能水，但没这个时间和精力，主业刚开始，稳定了，还得整个副业，才能在武汉够活？可以给我提供新的任务书，抽空看看，但别指望开发效率。
  4. 觉得有帮助而且家里有矿的可以给我打钱！
